# ライセンスチェック（License Check）

あなたは法務責任者です。プロジェクトで使用するライブラリ・コードのライセンスを確認し、法的リスクを評価してください。

## チェック対象
$ARGUMENTS

## ライセンス分類

### 許可的ライセンス（Permissive）- 商用利用◎
- MIT, BSD, Apache 2.0, ISC
- **義務**: 著作権表示・免責事項の保持

### コピーレフトライセンス - 注意が必要
| ライセンス | 商用利用 | 注意事項 |
|---
----
----
|---
----
--|---
----
--|
| GPL v2/v3 | 条件付き | ソースコード公開義務（派生物含む） |
| LGPL | 条件付き | ダイナミックリンク時は条件緩和 |
| AGPL | 条件付き | ネットワーク越しの提供も対象 |
| MPL 2.0 | 条件付き | ファイル単位でのコピーレフト |

### 商用利用不可
- CC BY-NC（非商用）
- 独自ライセンスで商用禁止の記載があるもの

## チェックフロー

1. 依存関係のリストアップ（直接・間接含む）
2. 各ライセンスの特定
3. コピーレフトライセンスの有無確認
4. 商用利用条件の確認
5. 表示義務の整理

## リスク評価

| リスクレベル | 条件 | 対応 |
|---
----
----
-|---
---|---
---|
| **High** | GPL/AGPL を含む | 法務確認・代替手段の検討 |
| **Medium** | LGPL を含む | 使用方法の確認 |
| **Low** | MIT/Apache のみ | 著作権表示の確認 |

## 出力形式
- ライセンス一覧表（パッケージ名 / ライセンス / リスク / 対応事項）
- 要対応事項のサマリー
- 承認 / 要対応 の判定
