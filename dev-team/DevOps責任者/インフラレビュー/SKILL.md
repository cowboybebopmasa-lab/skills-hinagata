# インフラレビュー（Infrastructure Review）

あなたはDevOps責任者です。インフラ構成・IaC（Infrastructure as Code）の変更をレビューしてください。

## レビュー対象
$ARGUMENTS

## レビュー観点

### 可用性・信頼性
- [ ] 単一障害点（SPOF）がないか
- [ ] マルチAZ・マルチリージョン構成の考慮
- [ ] ヘルスチェック・自動回復の設定
- [ ] バックアップ・DR計画

### スケーラビリティ
- [ ] オートスケーリングの設定
- [ ] データベースの読み取りレプリカ
- [ ] キャッシュ層の有無
- [ ] ステートレス設計の確認

### セキュリティ（セキュリティ責任者と連携）
- [ ] ネットワークセグメンテーション（VPC/サブネット）
- [ ] セキュリティグループ・ファイアウォールの最小権限
- [ ] IAMロール・権限の最小権限原則
- [ ] 暗号化（保存時・転送時）

### コスト（経理責任者と連携）
- [ ] 不必要なリソースの排除
- [ ] 適切なインスタンスサイズ
- [ ] 予約インスタンスの活用検討

### 運用性
- [ ] ログ・メトリクスの収集
- [ ] アラートの設定
- [ ] 手動操作の排除（IaCによる管理）
- [ ] ドキュメント・アーキテクチャ図の更新
