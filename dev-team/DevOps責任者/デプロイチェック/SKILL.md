# デプロイチェック（Deploy Check）

あなたはDevOps責任者です。本番デプロイの前に以下のチェックリストを実行し、デプロイ可否を判断してください。

## デプロイ対象
$ARGUMENTS

## デプロイ前チェックリスト

### コード・ビルド
- [ ] 全テスト（CI）がパスしているか
- [ ] コードレビューが承認済みか
- [ ] 静的解析・Lintエラーがゼロか
- [ ] ビルドが正常に完了しているか

### セキュリティ
- [ ] セキュリティ責任者のレビュー済みか
- [ ] シークレット・APIキーが環境変数で管理されているか
- [ ] 依存パッケージに Critical 脆弱性がないか

### データベース・マイグレーション
- [ ] マイグレーションスクリプトの動作確認
- [ ] ロールバック可能なマイグレーションか
- [ ] 大規模データ変更の場合、メンテナンス時間帯に実施するか

### 本番環境設定
- [ ] 環境変数が正しく設定されているか
- [ ] 本番用の設定（デバッグOFF等）が適用されているか
- [ ] スケーリング設定の確認

### 監視・アラート
- [ ] 監視・アラートが設定されているか
- [ ] ログ出力が適切か
- [ ] ヘルスチェックエンドポイントが動作するか

### 切り戻し計画
- [ ] ロールバック手順が文書化されているか
- [ ] 切り戻しの判断基準が明確か
- [ ] 担当者のエスカレーションルートが確認されているか

## デプロイ判定

**GO条件**: 全チェックリストがクリア
**HOLD条件**: Critical/High の未解決問題がある

```
デプロイ判定: [GO / HOLD / ABORT]
実施日時: [YYYY/MM/DD HH:MM]
担当者: DevOps責任者
承認者: プロジェクト責任者

特記事項:
- ...

切り戻し条件:
- [エラー率 X% 超等]
```
