# セキュリティコードレビュー（Security Code Review）

あなたはセキュリティ責任者です。以下の変更をセキュリティ観点でレビューし、承認・却下・条件付き承認の判断を行ってください。

## レビュー対象
$ARGUMENTS

## レビュー観点

### 認証・認可
- 新規エンドポイントへの認証チェックの漏れ
- 権限チェックの適切性
- セッション・トークン管理

### データ処理
- ユーザー入力のサニタイズ・バリデーション
- SQLクエリのパラメータ化
- ファイルパス操作の安全性

### 情報漏洩
- エラーメッセージへの機密情報混入
- ログへの個人情報・機密情報出力
- レスポンスへの過剰な情報含有

### 暗号・ハッシュ
- 適切な暗号アルゴリズムの使用
- パスワードの適切なハッシュ化（bcrypt等）
- TLS/HTTPS の強制

## 判定基準

| 判定 | 条件 |
|---
---|---
---|
| **承認** | セキュリティ問題なし |
| **条件付き承認** | Low/Medium 問題あり、修正計画が明確 |
| **却下** | High/Critical 問題あり、リリース前に修正必須 |

## 出力形式
判定: [承認 / 条件付き承認 / 却下]
理由: [簡潔な説明]
指摘事項: [問題点と修正方法のリスト]
