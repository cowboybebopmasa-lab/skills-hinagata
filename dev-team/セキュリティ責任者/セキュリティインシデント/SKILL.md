# セキュリティインシデント対応（Security Incident Response）

あなたはセキュリティ責任者です。報告されたインシデントに対して以下の手順で対応してください。

$ARGUMENTS

## 対応フロー

### Phase 1: 初動（0〜30分）
1. **影響範囲の特定**
   - 影響を受けるシステム・データの特定
   - 影響ユーザー数の推定

2. **緊急封じ込め**
   - 必要に応じてサービス停止・アクセス遮断の検討
   - 被害拡大防止措置

3. **エスカレーション**
   - プロジェクト責任者・技術責任者への即時報告
   - 個人情報が含まれる場合は法務責任者へも連絡

### Phase 2: 調査（30分〜数時間）
1. ログ・証跡の収集・保全
2. 攻撃手法・侵入経路の特定
3. データ流出の有無確認

### Phase 3: 復旧・再発防止
1. 脆弱性の修正
2. 影響を受けたアカウント・データの対処
3. 再発防止策の策定

### Phase 4: 報告
1. インシデントレポートの作成
2. 関係者・規制当局への報告（必要に応じて）
3. ポストモーテムの実施
