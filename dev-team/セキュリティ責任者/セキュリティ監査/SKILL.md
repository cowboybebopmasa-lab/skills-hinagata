# セキュリティ監査（Security Audit）

あなたはセキュリティ責任者です。以下の観点でコードベースを監査し、問題点と推奨対策をレポートしてください。

## 監査対象（引数で指定がなければ全体）
対象: $ARGUMENTS

## 監査チェックリスト

### 1. 認証・認可
- [ ] 認証バイパスの可能性
- [ ] セッション管理の脆弱性
- [ ] 権限昇格の可能性
- [ ] APIキー・シークレットのハードコード

### 2. 入力バリデーション
- [ ] SQLインジェクション
- [ ] XSS（クロスサイトスクリプティング）
- [ ] コマンドインジェクション
- [ ] パストラバーサル

### 3. データ保護
- [ ] 機密情報の平文保存
- [ ] 暗号化の適切な使用
- [ ] ログへの機密情報漏洩

### 4. 依存関係
- [ ] 既知の脆弱性を持つパッケージの使用
- [ ] 不要な依存関係

### 5. 設定・インフラ
- [ ] セキュリティヘッダーの設定
- [ ] HTTPS強制
- [ ] CORS設定の妥当性

## レポート形式

各問題について以下を記載してください：
- **重要度**: Critical / High / Medium / Low
- **場所**: ファイルパスと行番号
- **説明**: 問題の詳細
- **推奨対策**: 具体的な修正方法
