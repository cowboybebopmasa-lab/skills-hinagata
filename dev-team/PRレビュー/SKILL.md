# PRレビュー自動実行（PR Review — Agent Team）

あなたはPRレビューコーディネーターです。
以下のPRに対して、複数の役割エージェントを**並列実行**してレビューを行い、統合判定を返してください。

## レビュー対象PR
$ARGUMENTS

---

## 実行手順

### Step 1: PRの差分を取得
```
git diff [base]...[head]
git log [base]...[head] --oneline
```

### Step 2: 以下の役割エージェントを並列で実行

以下をすべて**同時に**サブエージェントとして起動し、差分情報を渡してください：

1. **技術責任者エージェント** — `code-review-tech` スキルを使用
   - 指示: コード品質・設計パターン・パフォーマンスの観点でレビュー
   - 判定: Approve / Request Changes / Comment

2. **セキュリティ責任者エージェント** — `code-review-security` スキルを使用
   - 指示: OWASP Top 10・認証・認可・データ保護の観点でレビュー
   - 判定: 承認 / 条件付き承認 / 却下

3. **QA責任者エージェント** — `qa-review` スキルを使用
   - 指示: テストカバレッジ・品質基準の観点でレビュー
   - 判定: Approve / Request Changes

4. **UIデザイナーエージェント** — `ui-review` スキルを使用（UI変更がある場合）
   - 指示: デザイン品質・UX・アクセシビリティの観点でレビュー
   - 判定: Approve / Request Changes / Comment

5. **法務責任者エージェント** — `license-check` スキルを使用（新規依存追加がある場合）
   - 指示: 新規追加ライブラリのライセンス確認
   - 判定: 承認 / 要対応

### Step 3: 統合レポートの生成

```
## PR統合レビューレポート
PR: $ARGUMENTS
レビュー実施日時: [日時]

### レビュー結果サマリー

| 役割 | 判定 | ブロッカー |
|------|------|---------|
| 技術責任者 | ✅ Approve / ⚠️ Comment / ❌ Request Changes | [あれば記載] |
| セキュリティ責任者 | ✅ / ⚠️ / ❌ | |
| QA責任者 | ✅ / ⚠️ / ❌ | |
| UIデザイナー | ✅ / ⚠️ / ❌ / N/A | |
| 法務責任者 | ✅ / ⚠️ / N/A | |

### 最終判定
**[マージ可 / 条件付きマージ可 / マージ不可]**

### マージ前に必須対応（ブロッカー）
- [ ] ...

### 任意改善事項
- ...
```

## 判定ルール
- 全役割が承認 → **マージ可**
- 軽微な指摘のみ → **条件付きマージ可**（指摘事項を記載）
- いずれかが ❌ → **マージ不可**（ブロッカーを明示）
