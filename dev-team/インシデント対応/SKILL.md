# インシデント緊急対応（Incident Response — Agent Team）

あなたはインシデント対応の指揮官です。
報告されたインシデントに対して関係エージェントを**即時並列起動**し、迅速な復旧を指揮してください。

## インシデント情報
$ARGUMENTS

---

## 即時実行（並列）

以下を**同時に**起動してください：

### 1. DevOps責任者エージェント（最優先）
- スキル: `deploy-check`, `monitoring-setup`
- タスク: サービス状態確認・緊急封じ込め・切り戻し判断
- 期限: 15分以内に初期報告

### 2. セキュリティ責任者エージェント
- スキル: `security-incident`
- タスク: セキュリティ起因の可能性確認・証跡保全
- 期限: 30分以内に評価

### 3. 技術責任者エージェント
- スキル: `code-review-tech`
- タスク: 直近のデプロイ・コード変更との関連調査
- 期限: 30分以内に調査結果

### 4. プロジェクト責任者エージェント
- スキル: `project-status`
- タスク: ステークホルダーへの状況報告・対外コミュニケーション
- 期限: 20分以内に第一報

### 5. 法務責任者エージェント（データ漏洩の疑いがある場合のみ）
- スキル: `compliance-review`, `privacy-review`
- タスク: 法的通知義務の確認
- 期限: 1時間以内に評価

---

## インシデント対応タイムライン

```
T+0分:   インシデント検知・インシデント対応開始
T+15分:  DevOps初期報告（影響範囲・暫定対処）
T+20分:  プロジェクト責任者が第一報をステークホルダーへ
T+30分:  全エージェントから初期評価レポート
T+60分:  原因特定・復旧計画策定
T+XX分:  サービス復旧
T+24h:   ポストモーテム実施
```
