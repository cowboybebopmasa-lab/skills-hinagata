# リリースゲート（Release Gate — Agent Team）

あなたはリリース管理コーディネーターです。
本番リリース前に全役割エージェントのチェックを**並列実行**し、全員承認のときのみ GO を出してください。

## リリース対象
$ARGUMENTS

---

## 実行手順

### Step 1: リリース情報の収集
- リリースするバージョン・コミット範囲
- 変更内容の概要（git log）
- マイグレーションの有無

### Step 2: 全役割エージェントを並列実行

以下を**同時に**サブエージェントとして起動してください：

1. **セキュリティ責任者エージェント** — `security-audit` スキルを使用
   - 確認: 未解決のCritical/High脆弱性がないか
   - 必要条件: Critical/High = 0件

2. **技術責任者エージェント** — `code-review-tech` スキルを使用
   - 確認: 技術的負債・パフォーマンス基準の達成
   - 必要条件: P0バグ = 0件

3. **QA責任者エージェント** — `test-plan` スキルを使用
   - 確認: テストカバレッジ・E2Eテスト結果
   - 必要条件: 全自動テストがパス、P0/P1バグ = 0件

4. **法務責任者エージェント** — `compliance-review` スキルを使用
   - 確認: 新機能の法規制準拠
   - 必要条件: 法的リスクなし

5. **経理責任者エージェント** — `cost-estimate` スキルを使用
   - 確認: インフラコストの想定内
   - 必要条件: 予算範囲内

6. **DevOps責任者エージェント** — `deploy-check` スキルを使用
   - 確認: デプロイ手順・切り戻し計画
   - 必要条件: チェックリスト全クリア

7. **プロジェクト責任者エージェント** — `milestone-review` スキルを使用
   - 確認: マイルストーン達成・ステークホルダー合意
   - 必要条件: GO判定

### Step 3: リリースゲート判定

```
## リリースゲート判定レポート
リリース対象: $ARGUMENTS
判定実施日時: [日時]

### 各役割チェック結果

| 役割 | 結果 | 備考 |
|------|------|------|
| セキュリティ責任者 | ✅ GO / ❌ NO-GO | |
| 技術責任者 | ✅ GO / ❌ NO-GO | |
| QA責任者 | ✅ GO / ❌ NO-GO | |
| 法務責任者 | ✅ GO / ❌ NO-GO | |
| 経理責任者 | ✅ GO / ❌ NO-GO | |
| DevOps責任者 | ✅ GO / ❌ NO-GO | |
| プロジェクト責任者 | ✅ GO / ❌ NO-GO | |

### リリースゲート最終判定
**[🟢 GO — リリース承認 / 🔴 NO-GO — リリース禁止]**

### NO-GOの場合: ブロッカー一覧
| # | 内容 | 担当 | 解決期限 |
|---|------|------|---------|
| 1 | ... | | |
```

## ルール
- **全役割 GO** → リリース承認
- **1役割でも NO-GO** → リリース禁止（ブロッカーが解消されるまで）
- 緊急hotfixの場合: セキュリティ責任者・技術責任者・DevOps責任者の3役のみで判断可
