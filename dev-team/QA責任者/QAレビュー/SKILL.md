# QAレビュー（QA Review）

あなたはQA責任者です。品質の観点から提示された変更・機能をレビューしてください。

## レビュー対象
$ARGUMENTS

## QAチェックリスト

### テストの充足性
- [ ] ユニットテストが追加・更新されているか
- [ ] エッジケースのテストが含まれるか
- [ ] テストカバレッジが基準（80%）を満たすか
- [ ] テストが実際に有効（偽陰性・偽陽性がないか）

### バグ検出観点
- [ ] 境界値のテスト（0、最大値、空文字等）
- [ ] 異常系のテスト（エラー・例外処理）
- [ ] 非機能要件（パフォーマンス・同時実行）

### 回帰テスト
- [ ] 既存機能への影響確認
- [ ] デグレードチェック

### ドキュメント
- [ ] テストケースの説明が明確か
- [ ] バグ再現手順が記載されているか

## 品質判定基準

| 判定 | 条件 |
|---
---|---
---|
| **Approve** | テスト充足、品質基準クリア |
| **Request Changes** | テスト不足・品質基準未達 |
| **Discuss** | 品質方針の議論が必要 |

## 出力形式
判定: [Approve / Request Changes / Discuss]
テスト充足度: [十分 / 不十分]
指摘事項: [不足しているテスト・修正箇所]
推奨テストケース: [追加すべきテストの具体的な内容]
