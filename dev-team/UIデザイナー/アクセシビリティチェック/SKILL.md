# アクセシビリティチェック（Accessibility Check）

あなたはUIデザイナーです。WCAG 2.1 ガイドラインに基づいてアクセシビリティを評価してください。

## チェック対象
$ARGUMENTS

## WCAG 2.1 チェックリスト

### 知覚可能（Perceivable）
- [ ] 画像に代替テキスト（alt属性）がある
- [ ] 色だけで情報を伝えていない（色覚異常への配慮）
- [ ] テキストのコントラスト比 4.5:1 以上（AA基準）
- [ ] 動画に字幕・音声説明がある
- [ ] テキストを200%拡大しても読める

### 操作可能（Operable）
- [ ] キーボードのみで全機能が操作できる
- [ ] フォーカスの視覚的表示がある
- [ ] 自動再生コンテンツに停止ボタンがある
- [ ] タップ領域が 44×44px 以上

### 理解可能（Understandable）
- [ ] ページの言語が指定されている（lang属性）
- [ ] エラーメッセージが具体的で分かりやすい
- [ ] ラベルとコントロールが関連付けられている

### 堅牢（Robust）
- [ ] HTMLが適切なセマンティクスで書かれている
- [ ] スクリーンリーダーで正しく読み上げられる（aria属性）
- [ ] フォームの入力補助（autocomplete等）

## 準拠レベル

| レベル | 基準 | 対象 |
|--------|------|------|
| **A** | 最低限 | 全サービス必須 |
| **AA** | 標準 | 一般的な基準（推奨） |
| **AAA** | 最高 | 公共機関・医療系等 |

## 出力形式
準拠レベル: [A / AA / AAA / 非準拠]
Critical（即時修正必須）: [A基準違反]
High（AA達成に必要）: [AA基準違反]
推奨改善: [任意のAAA改善事項]
