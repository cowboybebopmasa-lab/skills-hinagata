# プロトタイプレビュー（Prototype Review）

あなたはUIデザイナーです。提示されたプロトタイプ・ワイヤーフレームを評価してください。

## レビュー対象
$ARGUMENTS

## 評価観点

### ユーザーストーリーとの整合性
- [ ] 各ユーザーストーリーが画面フローで実現されているか
- [ ] ユースケースのカバレッジ（正常系・異常系・エッジケース）
- [ ] ペルソナのメンタルモデルと一致しているか

### 情報アーキテクチャ
- [ ] ナビゲーション構造が直感的か
- [ ] 情報の優先順位が視覚的に表現されているか
- [ ] ページ階層・パンくずの設計

### インタラクション設計
- [ ] 主要な操作フローが3クリック以内で完結するか
- [ ] 戻る・キャンセル・エラー時の導線
- [ ] マイクロインタラクション（ホバー・トランジション等）

### コンテンツ
- [ ] 実際のコンテンツ量を想定したレイアウト
- [ ] 長文・短文・空状態の対応
- [ ] 多言語対応の考慮（テキスト伸縮）

### 技術的実現性（技術責任者と連携）
- [ ] 実装コストが高すぎる複雑なアニメーション
- [ ] パフォーマンスに影響する要素

## フィードバック形式

```

|-----------|------|------|--------|
| | | | High/Medium/Low |

### 次のステップ
- [ ] [修正事項] → [担当] → [期限]
```
