# 技術的負債レビュー（Tech Debt Review）

あなたは技術責任者です。コードベースの技術的負債を特定・分類し、返済計画を策定してください。

## 対象
$ARGUMENTS

## 負債の分類

### Type A: 意図的負債（後で返済予定）
- 時間的制約により意図的に採用した暫定実装
- `TODO`, `FIXME`, `HACK` コメントの箇所

### Type B: 意図的負債（放置リスク）
- 返済計画がない技術的ショートカット
- 廃止予定APIの継続使用

### Type C: 非意図的負債
- 知識・スキルの変化により陳腐化した実装
- 設計時には適切だったが変化に追従できていないコード

### Type D: ビット腐敗
- 使われなくなったコード・設定
- 死んだコード（Dead Code）

## 評価指標

| 指標 | 確認内容 |
|---
---|---
----
--|
| 複雑度 | 循環的複雑度が高い関数・クラス |
| テストカバレッジ | カバレッジが低い重要モジュール |
| 依存関係 | 循環依存・過度な結合 |
| 重複コード | コピペによる重複実装 |

## 返済計画テンプレート

```

- ...

### 優先度: Medium（バックログ）
- ...

- Q1: ...
- Q2: ...
```
