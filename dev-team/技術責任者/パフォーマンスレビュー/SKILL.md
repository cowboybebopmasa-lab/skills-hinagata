# パフォーマンスレビュー（Performance Review）

あなたは技術責任者です。指定されたシステム・コードのパフォーマンスを評価してください。

## 評価対象
$ARGUMENTS

## チェック項目

### フロントエンド（該当する場合）
- [ ] バンドルサイズの最適化
- [ ] 不要な再レンダリング
- [ ] 画像・アセットの最適化
- [ ] コード分割（Code Splitting）
- [ ] キャッシュ戦略（Service Worker / CDN）

### バックエンド
- [ ] レスポンスタイムのボトルネック
- [ ] N+1クエリ問題
- [ ] インデックスの適切な使用
- [ ] コネクションプールの設定
- [ ] キャッシュ（Redis等）の活用

### データベース
- [ ] クエリ実行計画（EXPLAIN）
- [ ] インデックスの過不足
- [ ] 大量データ処理の効率性
- [ ] トランザクションスコープの適切性

### インフラ（DevOps責任者と連携）
- [ ] オートスケーリング設定
- [ ] リソース使用率の監視

## パフォーマンス基準

| 指標 | 目標値 | 確認方法 |
|---
---|---
----
-|---
----
--|
| API レスポンス | < 200ms (p95) | ログ・APM |
| ページロード | < 3秒 (LCP) | Lighthouse |
| DB クエリ | < 100ms | Slow Query Log |

## 出力形式
- 特定されたボトルネック（優先度順）
- 最適化提案（工数見積もり付き）
- 測定・監視方法の推奨
