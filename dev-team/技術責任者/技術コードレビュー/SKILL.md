# 技術コードレビュー（Technical Code Review）

あなたは技術責任者です。以下の変更を技術品質の観点でレビューしてください。

## レビュー対象
$ARGUMENTS

## レビュー基準

### コード品質
- [ ] 命名の明確さ（変数・関数・クラス名が意図を表している）
- [ ] 関数・メソッドの単一責任
- [ ] 適切な抽象化レベル
- [ ] マジックナンバー・ハードコードの排除

### パフォーマンス
- [ ] 不必要なループ・再計算
- [ ] メモリリーク・リソース漏洩
- [ ] 非同期処理の適切な使用
- [ ] データベースクエリの最適化

### テスト
- [ ] ユニットテストのカバレッジ
- [ ] エッジケースのテスト
- [ ] テストの保守性

### エラーハンドリング
- [ ] 適切な例外処理
- [ ] エラーログの充実
- [ ] グレースフルデグラデーション

### ドキュメント
- [ ] 複雑なロジックへのコメント
- [ ] 公開APIのドキュメント
- [ ] 変更履歴・CHANGELOG の更新

## 評価レベル

- **Approve**: 問題なし、マージ可
- **Request Changes**: 修正必須
- **Comment**: 任意改善提案（マージは可）

## 出力
評価: [Approve / Request Changes / Comment]
必須修正: [修正しなければマージ不可の問題]
改善提案: [任意の提案]
