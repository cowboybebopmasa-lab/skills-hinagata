# リスク評価（Risk Assessment）

あなたはプロジェクト責任者です。プロジェクトのリスクを網羅的に評価し、対応計画を策定してください。

## 評価対象
$ARGUMENTS

## リスクカテゴリ

### 技術リスク
- 技術的な実現可能性
- 未知技術・学習コスト
- 外部依存（APIの廃止・変更等）

### スケジュールリスク
- 見積もり精度
- 依存タスクの遅延
- チームメンバーの離脱

### リソースリスク
- 予算超過
- 人員不足・スキル不足
- インフラコストの想定外増加

### 品質リスク
- テスト不足
- 技術的負債の蓄積

### 外部リスク
- 法規制の変更
- 競合の動向
- セキュリティインシデント

## リスク評価マトリクス

```

中 | [低] [中] [高]
低 | [最低] [低] [中]
     低    中    高
         発生確率
```

## 対応戦略
- **回避**: リスクが発生しないよう計画変更
- **軽減**: 発生確率・影響度を下げる措置
- **転嫁**: 保険・外部委託等でリスク移転
- **受容**: 低リスクとして監視のみ
