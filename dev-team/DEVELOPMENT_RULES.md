# チーム開発ルール（Agent Teams Development Rules）

## バグ管理ルール

### 自動検出
- Bashコマンドが失敗した場合、`auto-bug-report.sh` フックが自動的に GitHub Issue を作成する
- Issue には重大度ラベル（critical / high / medium / low）と「自動検出」ラベルが付く

### 手動報告・管理スキル

| コマンド | 用途 |
|---------|------|
| `/バグ報告 [タイトル]\|[箇所]\|[エラー]\|[重大度]` | バグを手動でIssueに登録 |
| `/バグ解決 [Issue番号]\|[根本原因]\|[修正内容]\|[再発防止]` | バグ修正後にIssueをクローズ |
| `/バグ一覧` | オープン中のバグを重大度順に表示 |
| `/バグ管理` | バグ全体のサマリーを表示 |
| `/バグトリアージ [バグ内容]` | QA責任者による重大度・優先度評価 |

### 重大度別対応ルール

| 重大度 | 条件 | 対応期限 |
|--------|------|---------|
| critical | サービス停止・データ損失 | 即時（他作業を中断） |
| high | 主要機能の障害 | 今スプリント内 |
| medium | 一部機能に影響 | 次スプリント |
| low | 軽微な問題 | バックログ |

## 基本原則

1. **役割と責任の明確化**
   各エージェントは担当役割のスキルのみを行使し、他役割の判断を尊重する。

2. **承認フロー**
   - セキュリティ・法務に関わる変更は必ず **セキュリティ責任者** / **法務責任者** の承認を得る
   - アーキテクチャ変更は **技術責任者** の承認を得る
   - 予算・コストに影響する決定は **経理責任者** の確認を得る
   - リリース判断は **プロジェクト責任者** が最終決定する

3. **コードレビュー**
   - すべてのPRは最低1名のレビュー承認が必要
   - セキュリティ関連コードは **セキュリティ責任者** のレビュー必須
   - 外部ライブラリ追加は **法務責任者** のライセンス確認必須

4. **ドキュメント**
   - 重要な設計決定はADR（Architecture Decision Record）として記録する
   - APIの変更は必ず変更履歴に記録する
   - セキュリティインシデントは即時エスカレーション

5. **品質基準**
   - テストカバレッジ 80% 以上を維持する
   - Critical/High の脆弱性をリリースに含めない
   - パフォーマンス基準（レスポンスタイム等）を守る

## 役割一覧

| 役割 | スキルディレクトリ | 主な責任 |
|------|-------------------|---------|
| セキュリティ責任者 | `security-officer/` | セキュリティ監査・脆弱性管理 |
| 技術責任者 | `technical-officer/` | アーキテクチャ・コード品質 |
| 法務責任者 | `legal-officer/` | ライセンス・コンプライアンス |
| 経理責任者 | `accounting-officer/` | コスト・予算管理 |
| プロジェクト責任者 | `project-officer/` | 進捗・リスク管理 |
| QA責任者 | `qa-officer/` | テスト・品質保証 |
| DevOps責任者 | `devops-officer/` | インフラ・デプロイ管理 |

---

## コミット・ブランチルール

- **ブランチ命名**: `feature/`, `fix/`, `hotfix/`, `refactor/`, `docs/` のプレフィックスを使用
- **コミットメッセージ**: Conventional Commits形式（`feat:`, `fix:`, `docs:`, `chore:`, `security:` 等）
- **mainブランチへの直接プッシュ禁止**: 必ずPR経由でマージ

---

## エスカレーション基準

| 事象 | エスカレーション先 |
|------|----------------|
| セキュリティ脆弱性発見 | セキュリティ責任者（即時） |
| 法的リスクの発見 | 法務責任者（即時） |
| 予算超過の見込み | 経理責任者・プロジェクト責任者 |
| スケジュール遅延 | プロジェクト責任者 |
| 本番障害 | DevOps責任者・技術責任者（即時） |
